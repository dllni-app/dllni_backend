# API Contract: Supermarket Owner Top Selling Products

## Overview

- Module: `Supermarket`
- Audience: Supermarket Owner App
- Purpose: Provide the highlighted dashboard performance block data (`topProducts`, `offersImpact`, `bestOfferPerformance`) for the authenticated store owner.

---

## Endpoint

- Method: `GET`
- URL: `/api/v1/store-owner/dashboard/top-selling-products`
- Auth: `Bearer token (Sanctum)`
- Middleware: `auth:sanctum`

---

## Store Scope

- Store is resolved from authenticated user via `StoreOwnerContextService::ownedStore()`.
- Client should not send `storeId` in query/body.
- Response `supermarket.id` is the owner default store id.

---

## Query Parameters

| Name | Type | Required | Allowed Values | Notes |
|---|---|---:|---|---|
| `range` | string | No | `today`, `week`, `month`, `year`, `custom` | Default is `today` |
| `from` | date (`Y-m-d`) | If `range=custom` | Any valid date | Inclusive start date |
| `to` | date (`Y-m-d`) | If `range=custom` | Any valid date and `>= from` | Inclusive end date |

Validation source: `StoreOwnerDashboardPerformanceRequest`.

---

## Field Definitions

### `range`
- `key`: applied range key.
- `from`: effective range start date.
- `to`: effective range end date.

### `supermarket`
- `id`: supermarket store id resolved from authenticated owner.
- `name`: supermarket store name.

### `topProducts[]`
- `productId`: supermarket product id.
- `name`: product name.
- `quantity`: total quantity sold in selected range.
- `revenue`: total revenue from this product in selected range.

### `offersImpact`
- `ordersUsedOffers`: عدد الطلبات التي استخدمت عروض فعليًا.
- `utilizationRatePercent`: نسبة الاستفادة = `ordersUsedOffers / totalOrders * 100`.
- `offersRevenue`: إيرادات الطلبات التي استخدمت عروض.
- `discountedOrdersCount`: alias backward-compatible = `ordersUsedOffers`.
- `conversionRatePercent`: alias backward-compatible = `utilizationRatePercent`.
- `discountedRevenue`: alias backward-compatible = `offersRevenue`.
- `totalSavings`: total discount amount.

### `bestOfferPerformance`
- Can be `null` if no offer-linked orders exist in selected range.
- `offerId`: offer id.
- `name`: offer name.
- `offerType`: offer type (`sm_offers.offer_type`).
- `discountValue`: numeric offer discount value (nullable).
- `discountPercent`: numeric offer discount percent (nullable).
- `usesCount`: number of distinct orders that used this offer.
- `revenue`: total revenue generated by those orders.
- `totalSavings`: total discounts for those orders.

---

## Error Responses

### `401 Unauthorized`

```json
{
  "message": "Unauthenticated."
}
```

### `403 Forbidden` (module mismatch)

```json
{
  "message": "This endpoint is for supermarket sellers only."
}
```

### `403 Forbidden` (owner has no linked store)

```json
{
  "message": "No store found for the authenticated store owner."
}
```

### `422 Unprocessable Entity` (validation)

```json
{
  "message": "The given data was invalid.",
  "errors": {
    "to": [
      "The to field must be a date after or equal to from."
    ]
  }
}
```

## Example Requests

### Default (today)

```http
GET /api/v1/store-owner/dashboard/top-selling-products HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```

### Weekly

```http
GET /api/v1/store-owner/dashboard/top-selling-products?range=week HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```

### Custom Range

```http
GET /api/v1/store-owner/dashboard/top-selling-products?range=custom&from=2026-04-01&to=2026-04-27 HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```
