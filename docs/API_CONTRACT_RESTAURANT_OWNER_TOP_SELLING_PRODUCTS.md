# API Contract: Restaurant Owner Top Selling Products

## Overview

- Module: `Resturants`
- Audience: Restaurant Owner App
- Purpose: Provide the highlighted dashboard block data (`topProducts`, `offersImpact`, `bestOfferPerformance`) plus linked supermarket from authenticated owner.

---

## Endpoint

- Method: `GET`
- URL: `/api/v1/restaurant-owner/dashboard/top-selling-products`
- Auth: `Bearer token (Sanctum)`
- Middleware: `auth:sanctum`

---

## Query Parameters

| Name | Type | Required | Allowed Values | Notes |
|---|---|---:|---|---|
| `range` | string | No | `today`, `week`, `month`, `year`, `custom` | Default is `today` |
| `from` | date (`Y-m-d`) | If `range=custom` | Any valid date | Inclusive start date |
| `to` | date (`Y-m-d`) | If `range=custom` | Any valid date and `>= from` | Inclusive end date |

Validation source: `OwnerDashboardPerformanceRequest`.

---

## Field Definitions

### `range`
- `key`: applied range key.
- `from`: effective range start date.
- `to`: effective range end date.

### `supermarket`
- `id`: supermarket id resolved from authenticated owner via `RestaurantOwnerContext::supermarket()`.
- `name`: supermarket display name.

### `topProducts[]`
- `productId`: restaurant product id.
- `name`: product name.
- `quantity`: sold quantity in selected range.
- `revenue`: total revenue for that product in selected range.

### `offersImpact`
- `discountedOrdersCount`: count of orders with promo code or discount.
- `conversionRatePercent`: discounted orders / total orders * 100.
- `discountedRevenue`: total order amount of discounted orders.
- `totalSavings`: total discount amount.

### `bestOfferPerformance`
- Can be `null` if no promo performance exists in selected range.
- `promoCodeId`: promo code id.
- `code`: promo code string.
- `discountType`: promo discount type.
- `discountValue`: promo discount value.
- `usesCount`: number of orders that used promo.
- `revenue`: total revenue generated by promo usage.
- `totalSavings`: total discounts granted by promo usage.

---

## Error Responses

### `401 Unauthorized`

```json
{
  "message": "Unauthenticated."
}
```

### `403 Forbidden` (module mismatch)

```json
{
  "message": "This endpoint is for restaurant sellers only."
}
```

### `403 Forbidden` (owner has no linked restaurant)

```json
{
  "message": "No restaurant found for this owner."
}
```

### `403 Forbidden` (owner has no linked supermarket)

```json
{
  "message": "No supermarket found for this owner."
}
```

### `422 Unprocessable Entity` (validation)

```json
{
  "message": "The given data was invalid.",
  "errors": {
    "to": [
      "The to field must be a date after or equal to from."
    ]
  }
}
```

## Example Requests

### Default (today)

```http
GET /api/v1/restaurant-owner/dashboard/top-selling-products HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```

### Weekly

```http
GET /api/v1/restaurant-owner/dashboard/top-selling-products?range=week HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```

### Custom Range

```http
GET /api/v1/restaurant-owner/dashboard/top-selling-products?range=custom&from=2026-04-01&to=2026-04-27 HTTP/1.1
Authorization: Bearer {token}
Accept: application/json
```
